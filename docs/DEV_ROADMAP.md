# 天逆 — 开发蓝图

## 文档定位

本文档是《天逆》的**唯一执行级开发路线图**。它汇总全部 7 份设计文档中的系统模块，按优先级分层，标注依赖关系，并纳入计划中的本地 LLM 接入点。所有模块的状态在此追踪。

### 6.1 LLM 本地部署候选方案

| 方案 | 模型 | 内存占用 | 推理速度 | 适用 |
|------|------|----------|----------|------|
| **llama.cpp + Qwen2.5-1.5B-Q4** | 中文轻量 | ~1.2 GB | 手机 CPU 可跑 | 编年史/尸骸（段落级） |
| **MediaPipe LLM Inference** | Gemma 2B | ~1.5 GB | Android 优化 | 同上 + 对话 |
| **MLC LLM** | Qwen2.5-0.5B | ~500 MB | 极快 | 传闻润色/短文本 |
| **云端 API 降级** | 任意 | 0 | 快 | 预生成编年史打包进 assets |

**推荐路径**：初期用模板文本（不依赖 LLM）保证核心体验。P6 阶段先接 llama.cpp + Qwen2.5-1.5B 做编年史，尸骸叙事为第二步，对话润色为可选项。永远保留模板降级路径——LLM 不可用时不影响游戏运行。

## 十二、本地 LLM 接入时机

LLM 不应成为阻塞项。下图标注了 LLM 在各阶段的介入点和降级路径：

```
P0-P3（无 LLM）：
  所有叙事文本使用模板 → 不依赖 LLM 也能完整游玩

P4（世界引擎上线后）：
  传闻系统先用模板：「天元城王散修突破筑基」
  → P6 接入 LLM 后升级为：「天元城的王散修在灵石耗尽的那个晚上，
     对着空碗里最后一颗聚气丹看了很久，然后一口吞了下去。
```
