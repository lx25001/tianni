# 天逆 — 游戏总体规划

## 文档定位

本文档是「天逆」文字修仙游戏的总规划。详细数值设计见 `SYSTEMS.md`，物品系统见 `ITEM_SYSTEM.md`，洞府系统见 `CAVE_ABODE_DESIGN.md`。本文档回答"这个游戏到底有哪些系统、什么时候开放、互相之间怎么关联"。

---

## 总览：按境界解锁的系统

```
炼气期 ──────────────────────────────── 修炼/战斗/背包/主界地图
  │ 基础系统全开，玩家熟悉核心循环
  │
筑基期 ──────────────────────────────── 宗门入门/坊市交易
  │ 炼丹/炼器启蒙
  │
金丹期 ──────────────────────────────── 洞府建造/法宝槽解锁
  │ 功法融合（自创功法雏形）
  │
元婴期 ──────────────────────────────── 元婴分身/炼丹室/炼器室
  │ 秘境探索
  │
化神期 ──────────────────────────────── 阵法/神识大幅度扩展
  │ NPC 道侣系统
  │
合体期 ──────────────────────────────── 悟道室/道侣居/分府
  │ 宗门掌门竞争
  │
大乘期 ──────────────────────────────── 因果系统/天道感悟
  │ 飞升前置
  │
渡劫期 ──────────────────────────────── 天劫系统/转世
  │
飞升期+ ─────────────────────────────── 仙界内容（后期）
```

### 4.2 游历面板

「游历」是副本/远征入口——不同于主界的自由地图探索，游历是**结构化冒险**：

- 按当前角色境界，生成 3-5 个可选目的地（秘境/讨伐/遗迹）
- 每个目的地显示：名称、境界、危险度、可能掉落
- 点击进入 → 可能触发：战斗 / 采集 / 奇遇 / NPC 遭遇
- 游历结束 → 结算奖励 → 返回主界

## 十一、副业刚需化

### 11.1 定位

炼丹、炼器、阵法不是"可选副业"——是大后期生存的**绝对刚需**。

| 副业 | 刚需理由 |
|------|----------|
| 炼丹 | 道品丹药的唯一来源；解毒丹是消除重伤/理智下降的必需品 |
| 炼器 | 圣品/道品装备的唯一来源；NPC 坊市最高只卖珍品 |
| 阵法 | 洞府防御的唯一手段；无阵法 = 兽潮时洞府被毁 |

### 11.2 NPC 坊市基准价值体系

严格执行，控制玩家财富积累速度：

```
NPC 买入价 = 物品基准价值 × 0.4  （四折回收）
NPC 卖出价 = 物品基准价值 × 1.2  （溢价 20%）
```

坊市不出售珍品以上物品。后期顶级资源（道品、圣品）必须通过以下途径获取：
1. 深度经营 NPC 好感度（道侣/师父/挚友私赠）
2. 探索远古遗迹（一次性掉落）
3. 玩家亲手收集材料打造（炼丹/炼器）

### 11.3 天劫系统

渡劫期触发。天劫强度 = 境界 × 业力 × 道心。成功 → 飞升。失败 → 转世。

### 11.4 转世系统

渡劫失败或主动选择转世：
- 保留：部分记忆（功法名）、道心（不变）、灵根（不变）
- 重置：境界（→炼气 1 层）、背包（清空，部分物品可存「轮回戒」）
- 转世宿慧 buff：回到前世境界前修炼速度 +50%

### 11.5 因果系统

金仙以上解锁。因果线：
- 道侣：1.3× 修炼速度
- 宿敌：战斗伤害 +20%
- 师徒：悟性 +10
- 自创功法被 NPC 传抄：每 10 次 1.1×

### 11.6 死关系统

道心 ≥ 90 且主动选择「死关」离线。离线衰减反转，现实 1 天 = 游戏内 1000-5000× 修炼速度。

### 11.7 探索与秘境

| 功能 | 说明 |
|------|------|
| 秘境探索 | 限时开放的特殊地图 |
| 远古遗迹 | 珍-道品唯一掉落来源 |
| 藏宝图 | 坊市/NPC 获取，指向隐藏房间 |
| 游历 | 在地图上「游历」自动移动，触发随机事件 |

### 11.8 成就与天道碑

- 成就系统：首次突破/首次炼丹/首件仙品装备等
- 天道碑：本地记录玩家里程碑
- 奖励：成就点数可兑换稀有物品

### 11.9 天道排斥（反修改叙事彩蛋）

纯单机游戏不做封号/阻断。若系统定期扫描检测到属性或词条严重溢出正常上限，静默打上隐藏标签 `天道排斥·污浊`：

| 触发 | 效果 |
|------|------|
| `realm > 23` 或 `layer > 9` 且非正常突破 | 下次渡劫天雷威力 ×5-10 |
| 单件装备词条数 > 5 | 每超 1 条，该装备每日随机消去 1 个词条 |
| 背包物品数量 > capacity 且无扩容记录 | 每日随机物品消失（"虚空吞噬"） |
| 道心/根骨等属性 > 硬上限 ×2 | 触发顶级虚空风暴 + 杀手截杀事件 |

玩家不会被禁止游戏，但会被告知：

> "你的存在已引起天道的注意。天雷将比寻常猛烈数倍，虚空中的眼睛已经睁开。这是逆天而行的代价——若你准备好了，那就继续。"

将其包装为**高难度叙事彩蛋**，而非惩罚系统。

## 十三、数据持久化

### 13.1 技术选型

首选用对对象映射友好的轻量级本地数据库。两套方案对比：

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **Isar / Hive** | 原生 Dart，零 SQL，直接存 Dart 对象，查询速度快，支持索引和过滤 | Isar 已停止维护，Hive 对复杂关联查询弱 | 背包物品（JSON 复杂字段）、装备词条动态属性 |
| **SQLite（sqflite）** | 成熟稳定，关系型查询强，事务安全 | JSON 字段需手动序列化，高频查询需建关联表 | 角色数据、功法列表、NPC 好感度等结构化数据 |

**推荐方案**：Isar（或 Hive）存储背包/装备/洞府等复杂对象；SQLite 存储角色/功法/NPC 等结构化数据。如果希望统一为单一方案，优先使用 SQLite + JSON 索引表（将高频查询字段提取到独立列以避免解析整个 JSON blob）。

### 13.2 性能注意

- 后期 1350 格背包全满时，禁止在 UI 线程 parse 全量 JSON
- 按词条筛选装备 → 建立独立 `equipment_affixes` 索引表，避免扫全表
- 批量操作使用事务（`batch`），单次提交避免逐行写

### 13.3 SQLite 参考 Schema（结构化数据部分）

```sql
-- 角色主表
CREATE TABLE character (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  surname       TEXT NOT NULL,
  given_name    TEXT NOT NULL,
  root_element  TEXT NOT NULL,
  root_purity   TEXT NOT NULL,
  purity_rate   REAL NOT NULL,
  con           INTEGER DEFAULT 10,
  spi           INTEGER DEFAULT 10,
  qi            INTEGER DEFAULT 10,
  dao           INTEGER DEFAULT 50,
  ins           INTEGER DEFAULT 10,
  bon           INTEGER DEFAULT 10,
  realm_index   INTEGER DEFAULT 0,
  layer         INTEGER DEFAULT 1,
  xp            INTEGER DEFAULT 0,
  spirit_stones INTEGER DEFAULT 0,
  created_at    TEXT NOT NULL,
  updated_at    TEXT NOT NULL
);

-- 已学功法
CREATE TABLE learned_skill (
  id        INTEGER PRIMARY KEY AUTOINCREMENT,
  skill_id  TEXT NOT NULL,
  level     INTEGER DEFAULT 1,  -- 1入门 2熟练 3大成 4圆满
  equipped  INTEGER DEFAULT 0,
  FOREIGN KEY (skill_id) REFERENCES item_template(id)
);

-- 背包
CREATE TABLE inventory_slot (
  id       INTEGER PRIMARY KEY AUTOINCREMENT,
  item_id  TEXT NOT NULL,
  count    INTEGER NOT NULL DEFAULT 1,
  slot_idx INTEGER NOT NULL,  -- 0-based 槽位索引
  data     TEXT,              -- JSON: 装备词条/耐久等动态属性
  FOREIGN KEY (item_id) REFERENCES item_template(id)
);

-- 装备栏
CREATE TABLE equipped (
  id       INTEGER PRIMARY KEY AUTOINCREMENT,
  slot     TEXT NOT NULL,  -- weapon/armor/accessory/artifact
  item_id  TEXT NOT NULL,
  FOREIGN KEY (item_id) REFERENCES item_template(id)
);

-- 洞府
CREATE TABLE cave_abode (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  name          TEXT DEFAULT '未命名洞府',
  room_id       TEXT NOT NULL,     -- 地图房间 id
  spirit_vein   INTEGER DEFAULT 0, -- 灵脉等级 0-7（凡-道）
  training_room INTEGER DEFAULT 0,
  storage_level INTEGER DEFAULT 0,
  farm_level    INTEGER DEFAULT 0,
  alchemy_room  INTEGER DEFAULT 0,
  forge_room    INTEGER DEFAULT 0,
  formation     INTEGER DEFAULT 0,
  dao_room      INTEGER DEFAULT 0,
  created_at    TEXT NOT NULL
);

-- 洞府仓库
CREATE TABLE cave_storage (
  id       INTEGER PRIMARY KEY AUTOINCREMENT,
  item_id  TEXT NOT NULL,
  count    INTEGER NOT NULL DEFAULT 1,
  FOREIGN KEY (item_id) REFERENCES item_template(id)
);

-- 灵田（每块地一条记录）
CREATE TABLE farm_plot (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  plot_idx   INTEGER NOT NULL,  -- 0-15
  seed_id    TEXT,              -- NULL=空地
  planted_at TEXT,
  harvest_at TEXT,
  FOREIGN KEY (seed_id) REFERENCES item_template(id)
);

-- 洞府日志
CREATE TABLE cave_log (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  message    TEXT NOT NULL,
```
