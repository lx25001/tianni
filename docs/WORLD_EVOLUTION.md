## 一、初始世界基调：百废待兴

### 1.1 背景设定

游戏开局时的世界，并非名门正派高高在上、秩序井然的成熟修仙界，而是**天道刚刚崩碎不久、法则紊乱的乱世**。

> 一千年前，天道碎裂。没有人知道原因。
> 有人说是因为最后一个道祖妄图逆天，天道怒而自碎；
> 有人说天道本就有裂痕——道祖只是压垮它的最后一人。
> 没有人知道真相。但所有人都活在后果里。

这个世界现在叫**混沌纪元**。不是比喻。就是字面意思。

### 1.2 乱世的六大特征

| 特征 | 表现 | 对玩家影响 |
|------|------|-----------|
| **没有成熟宗门** | 开局时不存在太一道盟、铸剑城等稳定巨头。地图上多为流民营地、散修聚集地、残破遗迹 | 玩家无法开局就加入宗门——宗门还没诞生 |
| **资源狂暴** | 天道崩碎导致灵脉外溢。资源不是稀缺，而是狂暴且致命。采药可能触发灵气乱流 | 采集 / 炼丹自带风险，技能门槛更高 |
| **法外狂徒遍地** | 没有秩序，杀人越货是常态。坊市甚至没有安全交易区——全靠拳头和黑市 | 初期 NPC 平均罪恶值极高，友好 NPC 稀少 |
| **灵气紊乱** | 时辰加成随机波动 ±30%，灵脉浓度不稳定 | 修炼效率不稳定，找稳定灵脉是首要任务 |
| **妖兽狂暴** | 妖兽受天道碎片影响，攻击性 ×2，刷新率 ×3 | 野外极其危险，跨房间移动需谨慎 |
| **寿元焦虑** | 底层散修平均寿命极短——不是因为寿元上限低，是因为活不到那天 | 玩家会目睹大量 NPC 死亡 |

### 1.3 乱世的美学

乱世不是惩罚——是**叙事土壤**。

```
一个秩序井然的世界里，你见到一个金丹修士，只觉稀疏平常。
但在乱世里——
你见到一个金丹修士，你知道他从尸山血海里爬出来。
他身上的每一道疤、每一个眼神的闪烁，背后都有一段故事。
你敬他，不是因为他境界高。是因为他活下来了。
```

## 二、世界秩序值与三阶段演化

### 2.1 世界秩序值

底层设有一个隐藏的全局变量：**世界秩序值**（World Order），范围 0-100。

```
初始值：0
上限：100
增速：每游戏年 +0.1 ~ +2.0（取决于当年世界大事件）
```

秩序值不是「进度条」——它是**世界整体状态的一个快照指标**，反映当前世界的稳定程度。

### 2.2 影响秩序值的事件

| 事件 | 秩序变化 |
|------|----------|
| 一个 NPC 突破化神期 | +2 |
| 一个宗门正式建立（有山门 + 弟子 ≥ 10） | +5 |
| 一个宗门被灭 | -3 |
| 天道异变·天魔入侵 | -10 |
| 天道异变·灵气潮汐 | +3（资源充裕减少争夺） |
| 一个坊市建立安全交易区 | +1 |
| 万人以上的散修被妖兽潮屠杀 | -2 |

### 2.3 三阶段演化

```
┌─────────────────────────────────────────────────────────────┐
│                    世界演化三阶段                            │
│                                                             │
│  混沌期 (秩序 0-30)          拓荒期 (秩序 30-70)             │
│  ┌─────────────────┐        ┌─────────────────┐            │
│  │ · 群雄逐鹿       │        │ · 元婴/化神 NPC  │            │
│  │ · 高阶 NPC 忙于   │  ───→ │   建立雏形势力    │            │
│  │   抢占顶级灵脉    │        │ · 坊市出现安全区  │            │
│  │ · 底层散修死亡    │        │ · 宗门雏形出现    │            │
│  │   率极高         │        │ · 公告板活跃      │            │
│  │ · 黑市是唯一      │        │ · 正邪开始分化    │            │

```

### 2.4 各阶段对玩家的影响

**混沌期（秩序 0-30）**：
- 没有宗门可加入 → 玩家通过黑市、散修聚集地获取资源
- NPC 普遍不友好 → 社交需谨慎，好感度更难积累
- 资源狂暴但危险 → 高风险高回报
- 坊市无安全区 → 交易可能被抢
- **核心体验**：生存

**拓荒期（秩序 30-70）**：
- 宗门雏形出现 → 玩家可选择加入第一批宗门（创始元老身份）
- 安全交易区出现 → 经济系统启动
- 高阶 NPC 开始收徒 → 拜师系统激活
- **核心体验**：选择阵营

**宗门期（秩序 70-100）**：
- 稳定宗门体系 → 完整的宗门任务/贡献/藏经阁
- 正魔大战 → 大规模宗门战争
- 坊市成熟 → 拍卖行/鉴定/安全交易
- **核心体验**：在成熟秩序中成为顶尖

## 三、海量 NPC 金字塔架构

为了支撑「几万人在乱世中挣扎」的史诗感，底层代码彻底抛弃「全员实例化」。采用**三层 LOD（细节层级）架构**：

```
┌───────────────────────────────────────────────────────────┐
│                    NPC 金字塔                              │
│                                                           │
│                     ┌───────┐                              │
│                     │ T0 层  │  ← 玩家当前可见的 NPC        │
│                     │ ~10-30 │    完整实例化                │
│                     │        │    效用 AI 逐帧运行          │
│                     └───┬───┘                              │
│                         │                                  │
│                     ┌───┴───┐                              │
│                     │ T1 层  │  ← 已实体化但不在视野        │
│                     │ ~100-  │    JIT 惰性追溯              │
│                     │  200   │    有独立 ID/属性/因果       │
│                     │        │    不看就不算                │
│                     └───┬───┘                              │
│                         │                                  │
│                     ┌───┴───┐                              │
│                     │ T2 层  │  ← 宏观统计池（数据云）      │
│                     │ ~10000 │    纯数学公式                 │
│                     │   +    │    无独立 ID                 │
│                     │        │    仅记录数量/分布/死亡率     │
│                     └───────┘                              │
│                                                           │
│  T2 → T1 的唯一通道：实体化跃升（概率突破至筑基/金丹）      │
│  T1 → T0 的唯一通道：玩家走进 NPC 所在房间                  │
└───────────────────────────────────────────────────────────┘
```

## 四、T2 宏观统计层（数据云）

### 4.1 数据结构

T2 层不在 SQLite 中建立独立行。每个地图碎片（如「青云山碎片」）仅维护几个宏观变量：

```dart
class RegionalCloud {
  final String regionId;           // 地图碎片 ID
  List<int> lowLevelPool;          // 各小境界的散修数量 [炼气1层: 2000, 炼气2层: 1800, ...]
  int averageLayer;                // 平均境界层数
  double regionalWealth;           // 区域财富池（灵石总量）
  double fatalityRate;             // 区域死亡率（每月）
  double breakthroughRate;         // 区域突破率（每月出现筑基+ 的概率）
  double resourceDensity;          // 资源密度
  double dangerLevel;              // 危险等级（妖兽密度 × 灵气紊乱度）
  DateTime lastTickTime;           // 上次结算时间
}
```

### 4.2 宏观结算公式

每游戏月，执行一次纯数学推演：

```
// 月度死亡人数
monthlyDeaths = lowLevelPool.sum * fatalityRate * dangerLevel

// 从各层随机移出死亡人口（按比例）
for each layer in lowLevelPool:
    pool[layer] -= pool[layer] * fatalityRate * dangerLevel

// 月度突破人数（极小概率）
monthlyBreakthroughs = lowLevelPool.sum * breakthroughRate

// 突破者中：
//   95% 突破到下一小层（仍在 T2 中）
//   5% 突破到筑基/金丹 → 触发实体化跃升（进入 T1）
```

### 4.3 区域间流动

T2 散修不是静止的——他们会根据区域条件迁移：

```
迁移倾向：
  危险度高 → 流出 +30%
  资源密度高 → 流入 +20%
  有宗门建立 → 流入 +40%
  灵气紊乱 → 流出 +10%

迁移量 = 区域人口 × 0.05 × (1 + 净倾向值)
每月在相邻区域之间按迁移量调整各 lowLevelPool
```

### 4.4 玩家交互

玩家在 T2 区域采集/战斗时，系统用 `random` + `dangerLevel` 决定是否「遭遇散修」：

```
if (random.nextDouble() < dangerLevel * 0.1):
    生成一个临时炼气期 NPC（不持久化，战斗后回收）
    这代表「你撞上了一个在此地挣扎的散修」
```

## 五、实体化跃升（鲤鱼跃龙门）

### 5.1 触发条件

当宏观结算中，某个 T2 散修突破至**筑基期或金丹期**时，触发实体化跃升：

```
触发条件（OR 关系）：
  · 突破至筑基期（realmIndex = 1）
  · 突破至金丹期（realmIndex = 2）
  · 获得天品灵根（生成时极小概率）
  · 被 T1 高境 NPC 收为弟子（通过效用 AI 推演触发）
  · 玩家主动在 T2 区域「招募」散修（后期功能）
```

### 5.2 实例化过程

```
1. 系统调用 npc_factory.dart → 生成独立 NPC：
   · 从 T2 宏观池中 -1
   · 随机姓名（百家姓 + 名字生成器）
   · 随机性别/年龄
   · 随机 6 维主属性（5-18 区间）
   · 随机道心（30-70）
   · 随机灵根 + 纯度
   · 随机 5 维性格（见 UTILITY_AI_NPC.md §5）
   · 随机初始功法（按灵根匹配，至少 1 本）

2. 插入 SQLite 的 NPC 表 → 获得独立 ID

3. 加入因果网络：
   · 0-2 个随机因果关系（与现有 T1 NPC 建立）
   · 出生区域标记

4. 加入区域活跃 NPC 列表 → T1 层

5. 生成世界传闻：
   「尸山血海的青云山中，一名唤作赵铁柱的散修踩着同伴的尸骨结成了金丹。
     此人极为贪婪狠毒——青云山的风里都带着血腥味。」
```

### 5.3 实体化跃升的叙事意义

不是所有 NPC 都是「平衡分布」生成的。T2 跃升上来的 NPC 有独特的**背景故事**：

```
跃升 NPC 的隐含叙事：
  · 他来自 T2 数据云 → 他是从底层爬出来的
  · 他跃升时的区域条件（高死亡率/低资源）塑造了他的性格
    → 在乱世中活下来的人，通常不是好人
  · 他突破时的境界决定了起点——但能否走远，取决于后续推演
```

## 六、T1/T0 惰性追溯演算（JIT）

### 6.1 核心原则

**不看就不算。一看就追算。**

T1 层已实体化的 NPC，当玩家不在其所在区域时，不持续运行效用 AI。仅记录 `lastUpdateTime`。当玩家再次「关注」他时，瞬间追溯结算。

### 6.2 冻结与坍缩

```
冻结（每次玩家离开 NPC 所在房间时）：
  · 记录 npc.lastUpdateTime = now()
  · 记录 npc.lastSnapshot = {属性快照}
  · 从效用 AI 推演队列中移除

坍缩追溯（玩家再次关注时触发）：
  关注方式包括：
    · 玩家进入 NPC 所在房间
    · 玩家在天机阁查询该 NPC
    · 玩家通过因果网络感知（道侣/宿敌状态更新）
    · 洞府日志汇总

  追溯流程：
    1. Δt = now() - lastUpdateTime → 折算为游戏时辰数
    2. 如果 Δt < 1 个游戏时辰 → 无变化，直接回到 T0（玩家可见）
    3. 如果 Δt ≥ 1 个游戏时辰 → 启动追溯结算：
       
       ┌─────────────────────────────────────────┐
       │  岁月追溯公式（JIT 结算）                  │
       │                                          │
       │  每个游戏时辰推演 1 步（批量压缩版）：       │
       │                                          │
       │  · 修炼积累：修为 += 每秒修炼值 × 7200     │
       │            （7200 = 1 时辰的秒数）         │
       │  · 检查突破：修为满了？→ 尝试突破          │
       │  · 投掷事件骰：每个时辰 5% 触发随机事件     │
       │    → 采集/交易/战斗/社交（概率加权）       │
       │  · 投掷生存骰：每个游戏月 1% 概率判定      │
       │    → 渡劫失败/被仇家击杀/妖兽袭击          │
       │  · 投掷奇遇骰：每游戏年 0.5% 概率判定      │
       │    → 获得高品阶物品/功法/机缘              │
       │                                          │
       │  结果：                                   │
       │    80% 概率：NPC 还活着，属性有变化         │
       │    15% 概率：NPC 已死亡（在追溯中陨落）     │
       │    5% 概率：NPC 发生了重大事件              │
       │            （突破大境界/获得奇遇/结婚/被杀） │
       └─────────────────────────────────────────┘

    4. 如果 NPC 已死亡 → 清理因果线，生成墓碑数据
    5. 如果 NPC 还活着 → 更新属性，切换回 T0 层
```

### 6.3 性能收益

```
假设世界中有 150 个 T1 NPC，玩家在一个坊市停留（T0: ~10 个 NPC）：

  无惰性追溯：150 个 NPC 每时辰推演 × 在线时间 → 持续 CPU 消耗
  有惰性追溯：10 个 T0 NPC 实时推演 + 140 个冻结
              玩家离开坊市去另一个房间 → 当前 10 个冻结，新的 10 个解冻
              每个解冻的 NPC 追溯结算时间 Δt（通常 < 1 秒）

  手机 CPU 占用：
    实时推演部分：~10 个 NPC × 简单的效用评分 ≈ 忽略不计
    追溯结算部分：仅在 NPC 解冻时发生，单次 < 100ms

  零性能损耗——却能呈现「沧海桑田、物是人非」的沉浸感。
```

## 七、世界演化时序表

| 游戏时间 | 秩序值范围 | 典型事件 | 玩家体验 |
|----------|-----------|----------|----------|
| 第 1-5 年 | 0-10 | 乱世初期。每天数百散修死亡。第一个筑基 NPC 实体化跃升 | 纯粹的生存挑战。没有宗门、没有坊市安全区、全是黑市 |
| 第 5-20 年 | 10-30 | 第一个金丹 NPC 出现。第一个散修聚集地形成。黑市中间人 NPC 稳定存在 | 开始有 NPC 愿意交易/收徒。但信任成本极高 |
| 第 20-50 年 | 30-50 | 第一个元婴 NPC 突破。第一个宗门雏形建立。第一个坊市安全区出现 | 拓荒期。玩家可以加入最早期的宗门，成为元老 |
| 第 50-100 年 | 50-70 | 宗门扩张。正邪分化。第一次宗门战争。第一个化神 NPC 出现 | 完整的宗门体验。玩家已是「前辈」 |
| 第 100-200 年 | 70-90 | 天道异变首次触发。多个宗门并立。拍卖行系统成熟 | 宗门期。完整的修仙生态 |
| 第 200 年+ | 90-100 | 盛世。正魔大战。天道异变频率增加（世界太大，天道需要清理） | 玩家可能在此时飞升，或已成为宗门祖师 |

## 八、LLM 结合点：天道史官

若未来本地接入自然语言大模型，将其定位为「天道史官」——不参与推演逻辑，只负责**将干瘪的数据转化为可读的叙事文本**。

### 8.1 尸骸验尸录（微观）

在乱世地图中，玩家会经常遇到未被刷新的 NPC 尸体。玩家点击「搜魂 / 查验」时，将该实体的遗留标签发给 LLM：

```
输入数据（JSON）：
{
  "name": "王五",
  "realm": "炼气九层",
  "deathCause": "争夺百年朱果被焚烧",
  "inventory": ["半块烧焦的玉佩"],
  "location": "青云山南麓",
  "era": "混沌纪元 第 3 年"
}

LLM 输出（示例）：
「这具焦尸的手骨死死攥着那半块玉佩。搜魂残念里，浮现出一个画面——
  他不是为了自己突破才去夺那朱果。他只是想把朱果换成灵石，
  去坊市赎回被卖掉的道侣。火焰吞噬他的最后一个念头是：她的名字。
  那半块玉佩上刻着的，就是她的名字。」
```

### 8.2 编年史自动撰写（宏观）

每隔 10 个游戏年，系统将 T2 宏观数据的变化和 T1 实体 NPC 的大事件聚合打包，交给 LLM 撰写一卷《天逆编年史》：

```
输入数据（JSON）：
{
  "years": "混沌纪元 第 20-30 年",
  "orderChange": "秩序值 12 → 28",
  "majorEvents": [
    "青云山 T2 散修死亡率从 15% 降至 8%",
    "NPC 赵铁柱（金丹）在黑风谷建立血煞寨",
    "NPC 苏云清（元婴）在青云山建立青木寨雏形",
    "首次坊市安全交易区出现——由苏云清强制划定"
  ],
  "notableNPCs": ["赵铁柱", "苏云清"],
  "playerEvents": ["玩家 XXX 于第 25 年突破元婴"]
}

LLM 输出（示例）：
  《天逆编年史·第三卷》
  
  混沌纪元二十年至三十年，青云山脉的血腥味终于被风吹散了一些。
  
  这十年里，从尸山血海中爬出来的赵铁柱在黑风谷立了寨。没有人敢问他
  寨子为什么叫「血煞」——也没有人问那些在黑风谷失踪的散修去了哪里。
  
  但真正改变这片土地的人，是青木寨的苏云清。她在坊市划下了一条线。
  线的这边，可以交易，不可以杀人。违者，她亲自出手。
  这是这片土地上，第一次有人把「规则」这个词从天上拉回了人间。
  
  玩家 XXX 于此间突破元婴。天雷落了三日，青木寨的弟子们远远围观——
  他们中的大多数，从未见过一个人在雷劫中还能站着。
```

### 8.3 LLM 的边界

LLM 不参与推演——它不决定谁死谁活、谁突破谁失败。它只负责**在推演已经结束后，把它们讲成故事**。

```
不交给 LLM 的：
  ✗ NPC 战斗胜负
  ✗ 突破成功/失败
  ✗ 物品掉落
  ✗ 因果网络推演

可以交给 LLM 的：
  · T2 数据波动 → 一段乱世叙事
  · NPC 大事件 → 编年史段落
  · 玩家里程碑 → 写入编年史
  · 随机传闻的文本润色
  · NPC 对话的个性化表达（可选，仅在关键对话中）
```
